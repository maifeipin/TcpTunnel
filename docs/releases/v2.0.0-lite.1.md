# v2.0.0-lite.1

## Summary
Lightweight release profile focused on practical home lab and SMB use cases with minimal operational complexity.

## Target Use Cases
- One Windows desktop via RDP
- One or more internal web consoles via HTTP or HTTPS
- One Linux host via SSH

## Scope Principles
- TCP only
- Single active client model
- Multiple proxy rules from one client config
- No hot reload requirement
- Keep one-click deployment experience

## Included Improvements
- JSON multi-proxy client configuration
- Dynamic server-side multi-port listener registration
- Legacy compatibility for previous command-line startup
- Updated scripts and docs for easier rollout

## Recommended Default Ports
- RDP: 33890
- SSH: 2222
- HTTPS console: 8443

## Deployment Tips
1. Keep control and data ports restricted where possible.
2. Expose only the public ports actually needed.
3. Split each internal service into a dedicated proxy rule.
4. Use strong OS credentials and account lockout policies.

## Tag Purpose
Use this tag when you want the lightweight profile and stable baseline before adding any advanced features.
